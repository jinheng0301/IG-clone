# IG Clone

Professional Flutter Instagram clone project built with Firebase.

The actual Flutter app lives in `instagramzzz/`.

## Project Purpose

This repository is not just a UI exercise. It is a real-time social media clone built to practice production-style mobile engineering with:

- Flutter for cross-platform UI
- Firebase Authentication for login and account access
- Cloud Firestore for real-time social data
- Firebase Storage for uploaded media
- Provider for app state

This project should be developed with engineering discipline, not vague "vibe coding". AI can help, but every change still needs clear intent, traceable reasoning, and verification.

## Main Features

- Responsive Instagram-style UI
- Email and password authentication
- Share posts with captions
- Display posts with captions
- Delete posts
- Like posts
- Comment on posts
- Like comments
- Delete comments
- Update like counts and comment counts
- Search users
- Follow and unfollow users
- Display user posts, followers, and following on the profile screen
- Sign out
- Real-time data updates

## Engineering Standard

Use AI like a professional engineer:

- Define the exact problem before generating code.
- Ask for focused help, not broad random rewrites.
- Verify file-by-file changes before accepting them.
- Run the app or tests after meaningful changes.
- Keep architecture, naming, and behavior consistent.
- Reject code that works "by luck" but is not understood.
- Prefer maintainable fixes over fast hacks.

Bad AI usage:

- "Make this better."
- "Fix everything."
- Accepting generated code without reading it.
- Changing multiple systems at once without validation.

Good AI usage:

- "Update the Android Gradle config to match current Flutter tooling."
- "Refactor this screen without changing behavior."
- "Explain why this Firebase auth flow fails."
- "Add a feature and list every file changed."
